# Internationalization (i18n) Guide

uMailServer supports multiple languages for all web interfaces.

## Supported Languages

- 🇺🇸 English (en) - Default
- 🇹🇷 Turkish (tr)
- (Add more as needed)

## Adding a New Language

### 1. Create Translation File

Create a new JSON file in the locales directory:

```bash
# For admin panel
touch web/admin/src/locales/de.json

# For account panel
touch web/account/src/locales/de.json

# For webmail
touch webmail/src/locales/de.json
```

### 2. Copy Structure

Copy the structure from `en.json` and translate all values.

### 3. Register the Language

Update the `useI18n` hook to include the new language:

```javascript
const translations = {
  en: () => import('../locales/en.json'),
  tr: () => import('../locales/tr.json'),
  de: () => import('../locales/de.json'), // Add this
};
```

### 4. Update Language Selector

Add the new language to the selector component:

```javascript
const languages = [
  { code: 'en', name: 'English', flag: '🇺🇸' },
  { code: 'tr', name: 'Türkçe', flag: '🇹🇷' },
  { code: 'de', name: 'Deutsch', flag: '🇩🇪' }, // Add this
];
```

## Using Translations in Components

### React/JSX (Admin Panel)

```jsx
import { useI18n } from '../hooks/useI18n';

function MyComponent() {
  const { t } = useI18n();

  return (
    <div>
      <h1>{t('dashboard.title')}</h1>
      <p>{t('common.loading')}</p>
      <button>{t('common.save')}</button>
    </div>
  );
}
```

### TypeScript/TSX (Account Panel)

```tsx
import { useI18n } from '../hooks/useI18n';

function MyComponent() {
  const { t } = useI18n();

  return (
    <div>
      <h1>{t('profile.title')}</h1>
      <p>{t('common.loading')}</p>
    </div>
  );
}
```

### With Parameters

```jsx
// Translation: "Welcome, {{name}}!"
<p>{t('welcome.message', { name: 'John' })}</p>
// Output: "Welcome, John!"
```

## Translation Keys Convention

### Structure

```
section.subsection.key
```

Examples:
- `dashboard.title`
- `auth.login`
- `common.save`
- `domains.confirmDelete`

### Common Keys

Available in all translation files:

- `common.save`
- `common.cancel`
- `common.delete`
- `common.edit`
- `common.create`
- `common.loading`
- `common.error`
- `common.success`

## Language Persistence

User's language preference is stored in:
- `localStorage` key: `umailserver-language` (admin) or `umailserver-account-language` (account)
- Falls back to browser language
- Falls back to English

## Contributing Translations

1. Fork the repository
2. Create a new branch: `feature/i18n-[language]`
3. Add translation files for all three web apps
4. Test the translations in the UI
5. Submit a pull request

## Translation Checklist

- [ ] All keys from `en.json` are translated
- [ ] No missing keys in any language
- [ ] Special characters are properly escaped
- [ ] Parameters ({{name}}) are preserved
- [ ] UI looks good with the new language (check length)
- [ ] Language selector updated
- [ ] Hook updated to load new translations
