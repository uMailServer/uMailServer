## System Requirements

### Minimum Requirements

- **OS**: Linux (Ubuntu 20.04+), Windows Server 2019+, or macOS 12+
- **CPU**: 2 cores
- **RAM**: 4 GB
- **Disk**: 20 GB SSD
- **Network**: Public IP with ports 25, 465, 587, 143, 993, 80, 443 available

### Recommended Requirements

- **CPU**: 4+ cores
- **RAM**: 8+ GB
- **Disk**: 100+ GB NVMe SSD
- **Network**: Dedicated IP with reverse DNS (PTR) record

### DNS Requirements

Before deploying, ensure you have:

1. **A Record**: `mail.example.com` → Your server IP
2. **MX Record**: `example.com` → `mail.example.com` (priority 10)
3. **PTR Record**: Your IP → `mail.example.com` (reverse DNS)
4. **SPF Record**: `v=spf1 mx ~all`
5. **DKIM**: Will be generated by uMailServer after setup
6. **DMARC**: `v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com`

## Installation

### Binary Installation

```bash
# Download the latest release
curl -L -o umailserver.tar.gz https://github.com/umailserver/umailserver/releases/latest/download/umailserver-linux-amd64.tar.gz
tar -xzf umailserver.tar.gz
sudo mv umailserver /usr/local/bin/
sudo chmod +x /usr/local/bin/umailserver

# Create data directory
sudo mkdir -p /var/lib/umailserver
sudo mkdir -p /etc/umailserver
```

### Docker Installation

```bash
# Pull the image
docker pull umailserver/umailserver:latest

# Run with docker-compose
cat > docker-compose.yml << 'EOF'
version: '3.8'
services:
  umailserver:
    image: umailserver/umailserver:latest
    container_name: umailserver
    restart: unless-stopped
    ports:
      - "25:25"      # SMTP
      - "465:465"    # SMTPS
      - "587:587"    # Submission
      - "143:143"    # IMAP
      - "993:993"    # IMAPS
      - "110:110"    # POP3
      - "995:995"    # POP3S
      - "8080:8080"  # HTTP API
    volumes:
      - ./data:/var/lib/umailserver
      - ./config:/etc/umailserver
    environment:
      - UMAILSERVER_SERVER_HOSTNAME=mail.example.com
EOF

docker-compose up -d
```

## Configuration

### Initial Setup

```bash
# Generate initial config
umailserver setup --config /etc/umailserver/umailserver.yaml

# Or manually create config
cat > /etc/umailserver/umailserver.yaml << 'EOF'
server:
  hostname: mail.example.com
  data_dir: /var/lib/umailserver

tls:
  acme:
    enabled: true
    email: admin@example.com
    provider: letsencrypt
    challenge: http-01

smtp:
  inbound:
    enabled: true
    port: 25
    bind: 0.0.0.0
    max_message_size: 50MB
    max_recipients: 100
    max_connections: 100
  submission:
    enabled: true
    port: 587
    bind: 0.0.0.0
    require_auth: true
    require_tls: true
  submission_tls:
    enabled: true
    port: 465
    bind: 0.0.0.0
    require_auth: true

imap:
  enabled: true
  port: 993
  bind: 0.0.0.0
  max_connections: 100

pop3:
  enabled: true
  port: 995
  bind: 0.0.0.0
  max_connections: 100

http:
  enabled: true
  port: 8080
  bind: 0.0.0.0

admin:
  enabled: true
  port: 8081
  bind: 127.0.0.1

security:
  max_login_attempts: 5
  lockout_duration: 15m
  jwt_secret: "change-this-to-32-char-secret"

logging:
  level: info
  format: json
  output: /var/log/umailserver/umailserver.log
  max_size_mb: 100
  max_backups: 10
  max_age_days: 30

metrics:
  enabled: true
  port: 9090
  bind: 127.0.0.1
  path: /metrics

domains:
  - name: example.com
    max_accounts: 100
    max_mailbox_size: 1GB
EOF
```

### Security Hardening

1. **Generate strong JWT secret**:
```bash
openssl rand -base64 32
```

2. **Set proper file permissions**:
```bash
sudo chmod 600 /etc/umailserver/umailserver.yaml
sudo chown -R umailserver:umailserver /var/lib/umailserver
```

3. **Configure firewall**:
```bash
# UFW (Ubuntu)
sudo ufw allow 25/tcp
sudo ufw allow 465/tcp
sudo ufw allow 587/tcp
sudo ufw allow 143/tcp
sudo ufw allow 993/tcp
sudo ufw allow 110/tcp
sudo ufw allow 995/tcp
sudo ufw allow 8080/tcp

# Restrict admin panel to localhost
sudo ufw deny 8081/tcp
```

## Running the Server

### Systemd Service

Create `/etc/systemd/system/umailserver.service`:

```ini
[Unit]
Description=uMailServer Email Server
After=network.target

[Service]
Type=simple
User=umailserver
Group=umailserver
ExecStart=/usr/local/bin/umailserver --config /etc/umailserver/umailserver.yaml
ExecReload=/bin/kill -HUP $MAINPID
Restart=always
RestartSec=5

# Security hardening
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/lib/umailserver /var/log/umailserver

# Resource limits
LimitNOFILE=65535
LimitNPROC=4096

[Install]
WantedBy=multi-user.target
```

Enable and start:
```bash
sudo systemctl daemon-reload
sudo systemctl enable umailserver
sudo systemctl start umailserver
sudo systemctl status umailserver
```

### Environment Variables

You can override config values using environment variables:

```bash
export UMAILSERVER_SERVER_HOSTNAME=mail.example.com
export UMAILSERVER_SECURITY_JWTSECRET="your-secret-key"
export UMAILSERVER_LOGGING_LEVEL=debug
```

## Distributed Tracing

uMailServer supports OpenTelemetry distributed tracing for monitoring request flow across SMTP, IMAP, and HTTP components.

### Configuration

Enable tracing in your config:

```yaml
tracing:
  enabled: true
  service_name: "umailserver"
  exporter: "otlp"              # Options: noop, stdout, otlp
  otlp_endpoint: "localhost:4317"  # OTLP collector endpoint
  environment: "production"
  sample_rate: 0.1              # Trace 10% of requests
  attributes:
    deployment: "primary"
    region: "us-east-1"
```

### Jaeger Integration

1. **Start Jaeger**:
```bash
docker run -d --name jaeger \
  -e COLLECTOR_OTLP_ENABLED=true \
  -p 16686:16686 \
  -p 4317:4317 \
  jaegertracing/all-in-one:latest
```

2. **Configure uMailServer**:
```yaml
tracing:
  enabled: true
  exporter: "otlp"
  otlp_endpoint: "localhost:4317"
  sample_rate: 1.0
```

3. **View traces**: Open http://localhost:16686

### OpenTelemetry Collector

For production, use OpenTelemetry Collector:

```yaml
# collector-config.yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317

exporters:
  jaeger:
    endpoint: jaeger:14250
    tls:
      insecure: true

service:
  pipelines:
    traces:
      receivers: [otlp]
      exporters: [jaeger]
```

### Performance Impact

Tracing overhead is minimal:
- Noop exporter: ~0% overhead
- Stdout exporter: ~1% overhead  
- OTLP exporter: ~5% overhead (async batching)

Use sampling in high-traffic environments:
```yaml
tracing:
  sample_rate: 0.01  # Trace 1% of requests
```

## Backup & Recovery

### Automated Backups

```bash
# Add to crontab
0 2 * * * /usr/local/bin/umailserver backup --output /backup/umailserver-$(date +\%Y\%m\%d).tar.gz
```

### Backup Script

```bash
#!/bin/bash
# /usr/local/bin/umailserver-backup.sh

BACKUP_DIR="/backup/umailserver"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=30

# Create backup
mkdir -p $BACKUP_DIR
umailserver backup --output $BACKUP_DIR/umailserver-$DATE.tar.gz

# Verify backup
if [ -f "$BACKUP_DIR/umailserver-$DATE.tar.gz" ]; then
    echo "Backup completed: umailserver-$DATE.tar.gz"
    
    # Clean old backups
    find $BACKUP_DIR -name "umailserver-*.tar.gz" -mtime +$RETENTION_DAYS -delete
else
    echo "Backup failed!" >&2
    exit 1
fi
```

### Restore from Backup

```bash
# Stop server
sudo systemctl stop umailserver

# Restore data
sudo tar -xzf umailserver-20240101.tar.gz -C /

# Fix permissions
sudo chown -R umailserver:umailserver /var/lib/umailserver

# Start server
sudo systemctl start umailserver
```

## Troubleshooting

### Common Issues

#### Cannot receive emails

1. Check DNS records: `dig MX example.com`
2. Verify port 25 is open: `telnet mail.example.com 25`
3. Check reverse DNS: `dig -x YOUR_IP`
4. Review logs: `journalctl -u umailserver -f`

#### TLS certificate issues

1. Check ACME status: `umailserver status`
2. Verify port 80 is open for HTTP-01 challenge
3. Check certificate: `openssl s_client -connect mail.example.com:465`

#### High memory usage

1. Check queue: `umailserver queue list`
2. Review connections: `netstat -an | grep :25 | wc -l`
3. Enable pprof: Access `/debug/pprof/` on admin port

### Log Locations

- **Application logs**: `/var/log/umailserver/umailserver.log`
- **System logs**: `journalctl -u umailserver`
- **Mail logs**: `/var/lib/umailserver/logs/`

### Debug Mode

```bash
# Run with debug logging
umailserver --config /etc/umailserver/umailserver.yaml --log-level debug

# Or set environment variable
export UMAILSERVER_LOGGING_LEVEL=debug
```

### Performance Tuning

1. **Increase file descriptors**:
```bash
ulimit -n 65535
```

2. **Kernel tuning** (Linux):
```bash
# /etc/sysctl.conf
net.ipv4.tcp_keepalive_time = 300
net.ipv4.tcp_keepalive_probes = 5
net.ipv4.tcp_keepalive_intvl = 15
net.core.somaxconn = 65535
```

3. **Database optimization**:
```yaml
database:
  sync: false  # Disable fsync for better performance (risk of data loss on crash)
```

## Production Checklist

Before going live:

- [ ] DNS records configured (A, MX, PTR, SPF, DKIM, DMARC)
- [ ] TLS certificates configured (Let's Encrypt or manual)
- [ ] Firewall rules applied
- [ ] Backup system configured
- [ ] Monitoring and alerting enabled
- [ ] Log rotation configured
- [ ] Rate limiting enabled
- [ ] JWT secret is strong (32+ chars)
- [ ] Admin panel restricted to localhost/VPN
- [ ] Tested email flow (send/receive)
- [ ] Tested authentication (IMAP/SMTP)
- [ ] Reviewed security settings
